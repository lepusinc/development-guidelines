# ルーティング

このドキュメントでは、Laravel アプリケーションにおけるルーティングに関するガイドラインを定めます。

## 1. 命名規則

- **[要修正]** URL はケバブケースで統一する（例：`/user-profiles`）。
- **[要修正]** URL に動詞を含めない。操作は HTTP メソッドで表現する（`GET /users` ○、`GET /getUsers` ×）。
- **[要修正]** すべてのルートに `->name()` でルート名を付ける。URL の文字列参照を避け、`route()` ヘルパー経由で参照する。

## 2. 設計・構造

- **[厳守]** ルートファイルにはルート定義のみ記述する。クロージャにビジネスロジックを書かない。クロージャを含むルートはルートキャッシュが無効になるため、本番パフォーマンスに影響する。
- **[要修正]** 1ルートに対して1つのコントローラーアクションを割り当てる（Invocable Controller または Resource Controller のアクションメソッド）。
- **[要修正]** 共通のミドルウェア・プレフィックス・名前空間を持つルートはグループ化してまとめる。
- **[推奨]** CRUD 操作は `Route::resource` / `Route::apiResource` を使い RESTful に統一する。

## 3. セキュリティ

- **[厳守]** 認証が必要なルートには漏れなく `auth` ミドルウェアを設定する。コントローラー内の認証チェックのみに依存しない。
- **[要修正]** 認可（ポリシー）の適用はコントローラー内のみに依存せず、ルートミドルウェアと組み合わせて多重防御する。
